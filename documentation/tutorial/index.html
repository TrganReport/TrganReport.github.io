    Trgan Report Csharp Tutorial

    This page covers all the options Trgan report provides,

    Create and Generate Report

    The first step to create a trgan report is by initializing a trgan report object, install TrganReport from visual studio nuget source or add the below lines in your .csproj file 

    1
2
3
    <ItemGroup>
    <PackageReference Include="TrganReport" Version="1.0.0" />
</ItemGroup>
    Once installed import the package, 

    1
    using TrganReport;

    Start the report by initializing an object for TrganHtmlReport by specifying the desired output path with filename and html extension,

    2
    TrganHtmlReport report = new ("reports/TrganReport.html");
    Once the TrganHtmlReport object is created you can create TrganConainters or TrganTests concurrently. For reports that follows module base use TrganContainers to organize them

    3
    TrganContainer container = report.CreateContainer(name:"Login Page Module",categories=["login"]);

    Both TrganContainer and TrganTest can take optional parameter string[]: categories, which may helpful in visualizing the category or label or tags of a container in the Exection Status table Category column.

    For container based reports, concurrent Test creation is also possible, sequence of test creations will be like,

    4
5
6
    TrganTest test1 = container.CreateTest(name:"Login in with valid credentials");
TrganTest test2 = container.CreateTest(name:"Login in with invalid credentials");
TrganTest test3 = container.CreateTest(name:"Login in with valid username");

    Once the test is created you can update its record by calling any status updating methods such as

        test.Pass();
    test.Fail();
    test.Skip();
    test.Log(status);
    Here the arugment status can be any value from the enum Status

    You can add optional steps inside each TrganTest to mark check points for tracing or steps like BDD. To create a step inside a test, 

    7
8
9
    test1.AddStep(Status.PASS,"Entered user name");
test1.AddStep(Status.PASS, "Entered password");
test1.AddStep(Status.PASS, "Clicked login");

    if the report is BDD style then, 

    7
8
9
    test.AddStep(Status.PASS, GherkinKeyword.Given, "Entered username");
test.AddStep(Status.PASS, GherkinKeyword.When, "Entered password");
test.AddStep(Status.PASS, GherkinKeyword.Then,"Clicked login");
